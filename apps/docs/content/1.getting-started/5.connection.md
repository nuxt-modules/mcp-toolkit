## Overview

Once your MCP server is running, you can connect AI assistants like Cursor, VS Code, and ChatGPT to use your tools, resources, and prompts.

The module also provides components and routes to help your users install your MCP server in one click.

## Connect Your IDE

### Cursor

Click the button below to add your local MCP server to Cursor:

::install-button{name="local-mcp" url="http://localhost:3000/mcp" ide="cursor"}
::

Or manually add it to your Cursor settings (`~/.cursor/mcp.json`):

```json [~/.cursor/mcp.json]
{
  "mcpServers": {
    "my-nuxt-app": {
      "url": "http://localhost:3000/mcp"
    }
  }
}
```

### VS Code

Click the button below to add your local MCP server to VS Code:

::install-button{name="local-mcp" url="http://localhost:3000/mcp" ide="vscode"}
::

Or manually add the server to your VS Code MCP configuration (`.vscode/mcp.json`):

```json [.vscode/mcp.json]
{
  "servers": {
    "my-nuxt-app": {
      "type": "http",
      "url": "http://localhost:3000/mcp"
    }
  }
}
```

::callout{icon="i-lucide-info" color="info"}
Replace `my-nuxt-app` with your project name and update the URL if you're using a custom route or port.
::

## InstallButton Component

The module provides an `InstallButton` component that you can use in your documentation to let users install your MCP server in one click.

### Supported IDEs

| IDE | Value | Status |
|-----|-------|--------|
| Cursor | `cursor` | Supported |
| VS Code | `vscode` | Supported |

### In Vue Templates

```vue
<template>
  <!-- Cursor (default) -->
  <InstallButton name="my-app" url="https://my-app.com/mcp" />

  <!-- VS Code -->
  <InstallButton name="my-app" url="https://my-app.com/mcp" ide="vscode" />

  <!-- stdio server (CLI tools) -->
  <InstallButton
    name="my-cli"
    :config="{ command: 'npx', args: ['-y', '@my/mcp-server'] }"
  />

  <!-- Custom label -->
  <InstallButton
    name="my-app"
    url="https://my-app.com/mcp"
    label="Add to Cursor"
  />
</template>
```

### In Markdown (MDC Syntax)

If you're using [Nuxt Content](https://content.nuxt.com), use the MDC syntax:

```md
<!-- Cursor (default) -->
::install-button
---
name: "my-app"
url: "https://my-app.com/mcp"
---
::

<!-- VS Code -->
::install-button
---
name: "my-app"
url: "https://my-app.com/mcp"
ide: "vscode"
---
::

<!-- With custom label -->
::install-button
---
name: "my-app"
url: "https://my-app.com/mcp"
label: "Add to Cursor"
---
::
```

### Props Reference

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `name` | `string` | required | Name of the MCP server |
| `url` | `string` | - | URL for HTTP/SSE transport |
| `config` | `object` | - | Full config for stdio transport |
| `ide` | `'cursor' \| 'vscode'` | `'cursor'` | Target IDE |
| `label` | `string` | Auto-generated | Button label |
| `showIcon` | `boolean` | `true` | Show the IDE icon |

### Customization

The component uses CSS classes that you can override:

```css
/* Override default styles */
.mcp-install-button {
  background-color: #your-brand-color;
  border-radius: 9999px;
}
```

Or use the slot for completely custom content:

```vue
<InstallButton name="my-app" url="https://my-app.com/mcp">
  Add to Cursor
</InstallButton>
```

## README Badge

For README files and documentation outside of Vue/Nuxt, the module provides server routes to generate badges.

### Badge Routes

The module exposes two routes:

| Route | Description |
|-------|-------------|
| `/mcp/badge` | Redirects to the IDE deeplink |
| `/mcp/badge.svg` | Returns a customizable SVG badge image |

### Basic Usage

Add this to your README:

```md
[![Install in Cursor](https://your-app.com/mcp/badge.svg)](https://your-app.com/mcp/badge)
```

This will display a badge that, when clicked, opens the IDE and installs your MCP server.

### VS Code Badge

```md
[![Install in VS Code](https://your-app.com/mcp/badge.svg?ide=vscode)](https://your-app.com/mcp/badge?ide=vscode)
```

### Both IDEs

```md
[![Install in Cursor](https://your-app.com/mcp/badge.svg)](https://your-app.com/mcp/badge)
[![Install in VS Code](https://your-app.com/mcp/badge.svg?ide=vscode)](https://your-app.com/mcp/badge?ide=vscode)
```

### Customization Options

| Parameter | Default | Description |
|-----------|---------|-------------|
| `ide` | `cursor` | Target IDE (`cursor` or `vscode`) |
| `label` | Auto-generated | Badge text |
| `color` | `171717` | Background color (hex without #) |
| `textColor` | `ffffff` | Text color (hex without #) |
| `borderColor` | `404040` | Border color (hex without #) |
| `icon` | `true` | Show IDE icon (`true` or `false`) |

### Custom Badge Examples

**Custom label:**

```md
[![Add to Cursor](https://your-app.com/mcp/badge.svg?label=Add%20to%20Cursor)](https://your-app.com/mcp/badge)
```

**Custom colors:**

```md
[![Install](https://your-app.com/mcp/badge.svg?color=0ea5e9&borderColor=0284c7)](https://your-app.com/mcp/badge)
```

**Without icon:**

```md
[![Install](https://your-app.com/mcp/badge.svg?icon=false)](https://your-app.com/mcp/badge)
```

::callout{icon="i-lucide-info" color="info"}
Replace `https://your-app.com` with your actual domain. The badge route uses the server name from your `mcp.name` config.
::

## Deeplink Formats

For reference, here are the deeplink formats used by each IDE:

### Cursor

```
cursor://anysphere.cursor-deeplink/mcp/install?name=SERVER_NAME&config=BASE64_CONFIG
```

The config is Base64-encoded JSON.

### VS Code

```
vscode:mcp/install?name=SERVER_NAME&config=URL_ENCODED_JSON
```

The config is URL-encoded JSON.
